📖 Notice d'utilisation

Bienvenue dans Comptes & Budget, votre outil de gestion bancaire personnelle. Cette notice vous guide à travers les principales fonctionnalités.

1. Premier démarrage

  1. Configurer le solde de départ : ouvrez Import & données et renseignez la date à laquelle vous commencez votre suivi (ex. 01/01/2025) et le solde que vous aviez en banque à cette date. Cette valeur sert de base pour calculer votre solde réel à toute date ultérieure.
  2. Importer vos relevés bancaires en CSV : bouton 📥 Importer du ruban, ou glisser-déposer un fichier CSV sur la zone d'import, ou saisie manuelle via ➕ Ajouter. L'app gère les CSV des banques françaises (BPCE, Crédit Mutuel, Crédit Agricole) au format Windows-1252, séparateur point-virgule, dates JJ/MM/AAAA.

2. Les onglets

📊 Tableau de bord (Bilan)

Vue d'ensemble avec les indicateurs clés, l'évolution mensuelle revenus/dépenses, la répartition des dépenses par catégorie et les listes top dépenses / sources de revenus / plus grosses dépenses. Le KPI « Solde bancaire réel (pointé) » donne le solde réel du compte : solde initial + uniquement les opérations pointées jusqu'à la fin de la période choisie.

📋 Opérations

Liste complète des transactions avec filtres (recherche, catégorie, type, sens, pointage). La colonne P permet de pointer chaque opération d'un simple clic (○ non pointée / ✔ pointée et vérifiée sur le relevé). Les colonnes Date opér. et Date valeur indiquent ⏱ en orange lorsqu'elles diffèrent (débit différé).

  • Double-clic sur une ligne → ouvre le formulaire de modification
  • Touche Suppr → supprime l'opération sélectionnée
  • Touche Inser → nouvelle opération • F2 → modifier

🎯 Budget

Définissez un budget mensuel par catégorie. Les barres de progression deviennent vertes (< 80 %), oranges (< 100 %) ou rouges (dépassement) selon votre consommation pour la période sélectionnée.

🏷️ Catégories

Vue par catégorie avec détail : la liste des catégories (nombre d'opérations et total), et les opérations détaillées de la catégorie sélectionnée. Le bouton « Déplacer tout vers… » permet de recatégoriser en masse toutes les opérations d'une catégorie.

🧠 Règles auto

Les règles automatisent la catégorisation des opérations futures. Trois façons d'en créer : cocher « Mémoriser » dans le formulaire d'une opération, le bouton ➕ Nouvelle règle, ou le bouton 🔧 Harmoniser du ruban (suggestions automatiques d'après les libellés).

🔮 Prévisionnel

Déclarez vos opérations récurrentes (loyer, abonnements, salaire…) en précisant la fréquence (hebdo, mensuelle, trimestrielle, annuelle) et la date de début. L'app calcule automatiquement les prochains mois de prévisions avec totaux recettes / dépenses / net.

3. Période et mode date

La barre Période filtre toutes les vues. Le sélecteur Date contrôle la chronologie :

ModeQuand l'utiliser
Date d'opérationVision budget : l'achat compte le jour où il a eu lieu
Date de valeurSolde réel : l'achat compte le jour où la banque débite

Important pour les cartes à débit différé : un achat fait fin mai peut n'être débité qu'en juin. Le mode « Date valeur » est nécessaire pour retrouver à l'euro près le solde de votre relevé.

4. Pointage et rapprochement

💡 Le pointage est essentiel pour vérifier que vos opérations correspondent bien à votre relevé bancaire (rapprochement bancaire).

Quand vous recevez votre relevé, cliquez sur la colonne P de chaque ligne présente sur le relevé. Seules les opérations pointées comptent dans le solde bancaire réel. Si tout est pointé, ce solde doit correspondre exactement à votre solde bancaire.

6. Utilisation mobile (Android) & synchronisation avec le PC

Cette page HTML fonctionne sur le navigateur d'un téléphone Android et hors-ligne (les graphiques sont intégrés, aucune connexion requise). Voici le circuit recommandé pour saisir en déplacement et tout retrouver dans l'application Python à la maison.

💡 Important : les données saisies sur le téléphone sont stockées dans le navigateur de ce téléphone (pas dans le fichier). OneDrive synchronise le fichier HTML, jamais ces données. Il faut donc passer par des fichiers (JSON / CSV) pour transférer d'un appareil à l'autre — sauf à activer la synchronisation automatique ci-dessous.

🔗 Synchronisation automatique (nouveau — v1.9.0)

Sur ordinateur avec Chrome ou Edge, l'app peut se synchroniser toute seule avec l'application Python via un fichier partagé placé dans OneDrive (comptes_sync.json).

  1. Lance l'application Python une fois : elle crée et tient à jour comptes_sync.json dans le dossier OneDrive (automatique).
  2. Ouvre cette page dans Chrome/Edge, puis clique le bouton « 🔄 Activer la sync » en bas à droite et choisis ce même comptes_sync.json (une seule fois ; le choix est mémorisé).

Ensuite, plus rien à faire : à chaque modification, l'app écrit le fichier ; à l'ouverture, au retour sur l'onglet et toutes les 20 s, elle le relit et fusionne.

🧠 Fusion par opération : pour chaque opération, la version la plus récente est conservée (rien de plus récent n'est écrasé), et les suppressions faites d'un côté sont propagées à l'autre. Les deux applications restent ainsi alignées sur les données les plus à jour.
⚠️ La sync automatique nécessite Chrome/Edge sur ordinateur (API « File System Access »). Sur Firefox/Safari ou Android, utilise le circuit manuel ci-dessous — l'import fusionne désormais (il ne remplace plus). Évite d'éditer les deux côtés totalement hors-ligne en même temps : OneDrive pourrait créer un fichier « copie en conflit » que la sync n'ouvre pas.

📲 Ouvrir l'app sur Android

  1. Place le fichier comptes_budget_v2.html sur le téléphone (via OneDrive, l'app Fichiers, ou un câble).
  2. Ouvre-le avec Chrome (appui long sur le fichier → « Ouvrir avec » → Chrome). Ajoute la page à l'écran d'accueil pour la retrouver facilement.

🔄 Avant le départ (PC → téléphone)

  1. Dans l'application Python, fais 💾 Exporter (JSON) et dépose le fichier dans OneDrive.
  2. Sur le téléphone, ouvre l'onglet Import & données → « Charger / fusionner une sauvegarde JSON » et choisis ce fichier. Les données du PC fusionnent avec celles du téléphone (la version la plus récente de chaque opération est conservée).

✍️ En déplacement

Saisis tes opérations normalement (➕ Ajouter). Tout est enregistré automatiquement dans le navigateur du téléphone.

🏠 De retour (téléphone → PC)

  1. Sur le téléphone : onglet Import & données → « ⬇ Export CSV nouvelles opérations → Python ». Le fichier CSV (uniquement ce que tu as saisi depuis le chargement) part dans Téléchargements → envoie-le vers le PC (OneDrive, mail…).
  2. Dans l'application Python : 📥 Importer CSV et sélectionne ce fichier.
⚠️ L'import CSV de Python ne reprend que les transactions : le pointage et les éventuelles règles/budgets/récurrences créés sur le téléphone ne sont pas transférés par cette voie. Pour une sauvegarde complète du téléphone, utilise plutôt « 💾 Exporter tout (JSON) ».

📚 Glossaire

Catégorie
Classement principal d'une opération (Alimentation, Transports, Logement…) utilisé pour les statistiques et le budget. Chaque opération a exactement une catégorie.
Date d'opération
Date à laquelle vous avez fait l'achat. C'est la date « budget » : utile pour savoir quand vous avez dépensé.
Date de valeur
Date à laquelle la banque débite (ou crédite) effectivement le compte. Pour une carte à débit immédiat, c'est la même que la date d'opération ; pour un débit différé, elle peut être plus tardive.
Débit différé
Mode de certaines cartes où les achats du mois sont regroupés et débités en une fois (souvent le 5 ou 6 du mois suivant). Reconnu par l'icône ⏱ orange dans la colonne Date valeur.
Doublon
Opération qui apparaît deux fois (même date, même montant, même début de libellé). L'outil 🔍 Doublons les détecte et propose de les supprimer.
Encours
Ensemble des opérations en attente de débit, typiquement les achats à débit différé pas encore prélevés par la banque.
Harmonisation
Outil qui propose automatiquement des catégorisations basées sur des motifs prédéfinis (ex : « Carrefour » → Alimentation). Suggestion ponctuelle, pas une règle persistante.
Libellé
Texte descriptif de l'opération tel qu'apparu sur le relevé (« CARREFOUR MARKET 5012 »…).
Motif
Texte qu'une règle cherche dans le libellé. Sensible à la longueur : CARREFOUR matche tous les Carrefour ; CARREFOUR MARKET 5012 ne matche que ce magasin précis.
Mouvement net
Somme algébrique des opérations sur la période : revenus moins dépenses.
Opération récurrente
Opération qui se répète à intervalle fixe (loyer, abonnement, salaire…). Définie dans Prévisionnel.
Pointage
Action de cocher une opération comme « vérifiée sur le relevé bancaire ». Symbolisée par ✔ dans la colonne P. Seules les opérations pointées comptent dans le solde bancaire réel.
Période
Filtre temporel appliqué aux vues : « Toutes », une année, ou un mois précis.
Rapprochement bancaire
Comparer ligne à ligne ses opérations enregistrées avec celles du relevé bancaire, via le pointage.
Règle automatique
Affectation automatique d'une catégorie aux opérations dont le libellé correspond à un motif. Appliquée à chaque import CSV.
Solde bancaire réel
Solde initial + somme des opérations pointées depuis la date initiale. KPI principal du Bilan.
Solde de départ (initial)
Valeur de référence du compte à une date donnée, saisie dans Import & données.
Sous-catégorie
Précision facultative à l'intérieur d'une catégorie (Alimentation › Restauration rapide…).
Taux d'épargne
Part des revenus non dépensée : mouvement net / revenus × 100.
Transaction exclue
Catégorie spéciale pour les opérations qui ne doivent pas compter dans les statistiques (virements internes, cumuls de débit différé…).
